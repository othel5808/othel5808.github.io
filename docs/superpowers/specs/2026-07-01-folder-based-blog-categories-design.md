# 폴더 기반 블로그 카테고리 설계

## 목표

블로그 카테고리 메뉴가 실제 글 목록을 필터링하도록 고치고, Markdown 파일이 들어 있는 첫 번째 하위 폴더를 카테고리로 자동 인식한다. 글을 추가할 때 카테고리 목록과 글 개수를 별도로 수정하지 않아도 정적 빌드 결과에 반영되어야 한다.

## 콘텐츠 구조

블로그 글은 Astro 콘텐츠 컬렉션으로 관리한다.

```text
src/content/blog/
├── backend/
│   └── notes-on-astro.md
├── project/
│   ├── building-a-personal-site.md
│   └── writing-better-case-studies.md
└── design/
    └── designing-document-navigation.md
```

컬렉션 엔트리 ID의 첫 번째 경로 조각을 카테고리 ID로 사용한다. 각 글의 frontmatter에는 제목, 게시일, 요약, 소개 문구, 태그, 읽기 시간, 새 글 표시 여부만 둔다. 카테고리는 frontmatter에 중복 작성하지 않는다.

카테고리 표시 이름은 작은 설정 객체에서 관리한다. 알려지지 않은 새 폴더는 폴더명을 표시 이름으로 사용하므로 콘텐츠 추가가 빌드를 막지 않는다.

## URL과 페이지

- `/blog/`: 모든 글을 최신순으로 표시한다.
- `/blog/[category]/`: 해당 카테고리 폴더의 글만 표시한다.
- `/blog/[category]/[slug]/`: Markdown 본문을 렌더링하는 글 상세 페이지다.

정적 빌드에서 콘텐츠 컬렉션을 읽어 `getStaticPaths()`로 카테고리와 글 상세 경로를 생성한다. 실제 폴더가 없는 카테고리 URL은 생성하지 않는다.

기존 `/blog/[slug]/` 주소는 현재 데모 단계이며 외부 링크 호환 요구가 없으므로 새 중첩 주소로 교체한다. 홈 화면과 관련 글 링크도 새 주소를 사용한다.

## 카테고리 사이드바

사이드바는 전체 콘텐츠에서 카테고리별 글 개수를 계산한다. 현재 URL에 따라 전체 글 또는 해당 카테고리 링크에 `active` 클래스를 적용한다.

데스크톱과 태블릿에서는 기존 고정 사이드바 동작을 유지한다. 휴대폰에서는 기존 카테고리 버튼으로 메뉴를 열고, 링크를 선택하면 메뉴를 닫은 뒤 카테고리 페이지로 이동한다.

## 글 목록과 관련 글

목록 페이지는 공통 컴포넌트 또는 공통 데이터 유틸리티를 사용해 전체 목록과 카테고리 목록의 마크업을 동일하게 유지한다. 게시일은 최신순으로 정렬한다.

관련 글은 현재 글을 제외하고 다음 점수로 정렬한다.

- 같은 카테고리: 1점
- 같은 태그 하나당: 2점

점수가 같은 경우 게시일이 최신인 글을 먼저 보여주고 최대 3개로 제한한다.

## 마이그레이션

현재 `src/data/posts.mjs`의 예시 글 네 개를 Markdown 파일로 옮긴다. 기존 상세 페이지에 하드코딩된 본문도 각 Markdown 본문으로 이동한다. 데이터 배열은 제거하고 콘텐츠 컬렉션을 유일한 글 원본으로 사용한다.

## 오류 처리

- 필수 frontmatter가 누락되거나 타입이 잘못되면 컬렉션 스키마 검증으로 빌드를 실패시킨다.
- 매핑되지 않은 카테고리 폴더는 폴더명 자체를 표시해 새 폴더 추가를 허용한다.
- 콘텐츠가 없는 카테고리는 경로와 사이드바 항목을 생성하지 않는다.

## 검증

자동 테스트는 다음을 확인한다.

1. 폴더 경로에서 카테고리와 slug를 추출한다.
2. 카테고리 개수와 필터 결과가 실제 콘텐츠와 일치한다.
3. 관련 글 계산에서 현재 글을 제외하고 카테고리·태그 점수를 반영한다.
4. 빌드 결과에 전체 목록, 카테고리 목록, 중첩 상세 경로가 생성된다.
5. 기존 쿼리 기반 카테고리 링크와 수동 `POSTS` 배열이 남지 않는다.

브라우저 검증은 데스크톱과 휴대폰 폭에서 카테고리 링크 이동, 활성 상태, 필터된 글 목록, 모바일 메뉴 열기·닫기를 확인한다.

## 범위 제외

검색, 페이지네이션, RSS, CMS 편집 화면, 카테고리 다단계 중첩은 이번 변경에 포함하지 않는다.
