# Astro 포트폴리오 블로그 설계

## 목표

GitHub Pages에서 운영할 개인 사이트를 Astro로 구축한다. 사이트의 중심은 포트폴리오이며, 상세한 기술 기록은 블로그로 축적하고 이력서는 별도의 문서형 화면으로 제공한다. 초기 콘텐츠는 교체하기 쉬운 예시 데이터로 채운다.

## 디자인 방향

- 참고 사이트 `taegon.kim`의 절제된 개발자 블로그 분위기와 읽기 쉬운 콘텐츠 계층을 참고한다.
- 시각 방향은 선택된 B안인 문서형 포트폴리오를 사용한다.
- 상단 내비게이션은 A안처럼 이름 또는 로고를 왼쪽에, `포트폴리오`, `글`, `이력서`, 검색을 오른쪽에 둔다.
- 포인트 색상은 따뜻한 오렌지를 사용하고, 본문은 밝은 중성 배경과 짙은 텍스트로 구성한다.
- 모노스페이스 글꼴은 번호, 날짜, 카테고리 등 메타데이터에만 제한적으로 사용한다.
- 다크 모드와 모바일 레이아웃을 지원한다.

## 정보 구조

### 홈

- 짧은 자기소개와 전문 분야
- 대표 프로젝트 일부
- 최신 글 일부
- GitHub, 이메일 등 연락 링크

### 포트폴리오

왼쪽 고정 목차와 긴 단일 문서형 본문을 사용한다.

1. 소개
2. 핵심 역량
3. 대표 프로젝트
4. 오픈소스 및 사이드 프로젝트
5. 경력 요약
6. 연락처

프로젝트 예시는 문제, 역할, 해결 방식, 결과, 사용 기술, 관련 링크를 포함한다. 초기에는 현실적인 샘플 데이터를 넣어 사용자가 구조를 쉽게 교체할 수 있게 한다.

### 글

- 데스크톱과 태블릿에서 왼쪽 전체 높이 사이드바로 표시되는 카테고리 목록
- 휴대폰에서 작은 버튼으로 접히고 플로팅 팝업으로 펼쳐지는 카테고리 목록
- 새 글에 대한 시각적 표시
- 제목, 요약, 게시일, 카테고리, 예상 읽기 시간
- 동적 라우트 기반 개별 글 페이지
- 개별 글 하단의 태그 목록과 관련 글 최대 3개
- 관련 글은 현재 글과 태그 및 카테고리가 겹치는 정도를 점수화해 자동 선택
- Markdown 또는 MDX 기반 콘텐츠 관리

### 이력서

포트폴리오와 같은 왼쪽 목차 패턴을 공유하되 더 조밀하고 공식적인 문서 스타일을 사용한다.

1. 요약
2. 경력
3. 주요 프로젝트
4. 기술
5. 교육
6. 활동

## 공통 구성 요소

- 공통 헤더와 모바일 메뉴
- 페이지 내부 목차 사이드바
- 프로젝트 카드 또는 프로젝트 요약 행
- 글 목록과 카테고리 칩
- 공통 푸터
- 테마 전환
- 검색 진입점
- SEO 및 소셜 공유 메타데이터
- RSS 피드

## 콘텐츠 구조

Astro Content Collections로 글과 프로젝트 데이터를 관리한다. 스키마 검증을 통해 제목, 설명, 날짜, 카테고리, 기술, 공개 여부 등의 필드를 일관되게 유지한다. 포트폴리오와 이력서의 긴 본문도 교체하기 쉬운 콘텐츠 파일로 분리한다.

## 라우팅과 배포

- `/`: 홈
- `/portfolio/`: 포트폴리오
- `/blog/`: 글 목록
- `/blog/[slug]/`: 개별 글
- `/resume/`: 이력서
- GitHub Actions에서 정적 빌드를 실행하고 GitHub Pages에 배포한다.
- 저장소 이름에 따른 기본 경로와 canonical URL을 Astro 설정에서 처리한다.

## 반응형 동작

- 포트폴리오는 혼합형 프레젠테이션 레이아웃을 사용한다. 소개와 핵심 역량은 화면 높이에 가까운 큰 슬라이드 섹션으로, 프로젝트는 16:9 케이스 스터디 카드로 표현한다.
- 태블릿과 데스크톱에서는 포트폴리오 목차를 화면 왼쪽 전체 높이의 고정 사이드바로 표시한다. 본문은 사이드바 너비만큼 이동해 겹치지 않는다.
- 고정 사이드바는 접기 버튼으로 화면 왼쪽의 얇은 레일까지 축소할 수 있고 다시 펼칠 수 있다.
- 휴대폰 너비에서만 포트폴리오 목차를 화면 좌측 하단의 작은 버튼으로 접는다. 버튼을 누르면 세로 메뉴가 펼쳐지고 항목 선택 후 다시 닫힌다.
- 이력서는 태블릿과 데스크톱에서 세로 목차를 유지하고 휴대폰에서만 접는다.
- 글 목록은 좁은 화면에서 단일 열이 되며 메타데이터 줄바꿈을 허용한다.
- 홈, 글, 이력서, 개별 글을 포함한 모든 페이지는 데스크톱, 태블릿, 휴대폰 너비를 지원한다.

## 접근성과 오류 처리

- 의미론적 HTML, 키보드 탐색, 명확한 포커스 표시를 제공한다.
- 색상 대비와 모션 감소 설정을 존중한다.
- 존재하지 않는 글은 정적 생성 대상에서 제외하고 사용자용 404 페이지를 제공한다.
- 필수 콘텐츠 필드 누락은 빌드 단계의 스키마 검증으로 차단한다.

## 검증

- Astro 정적 빌드 성공 여부
- 핵심 경로 생성 여부
- 콘텐츠 스키마 검증
- 데스크톱 및 모바일 레이아웃 시각 확인
- 키보드 탐색과 명암 모드 확인
- GitHub Pages 기본 경로에서 자산과 내부 링크가 정상 동작하는지 확인

- CMS, 관리자 화면, 댓글, 회원 기능
- 서버 검색이나 외부 검색 서비스
- 실제 개인 경력과 프로젝트 원고 작성
- 브레인스토밍용 `.superpowers` 목업 파일의 배포
